## Part 1. 임베디드 시스템 개요

### Q1. MCU와 임베디드 시스템의 차이

MCU(마이크로컨트롤러)는 마이크로프로세서(ALU, Register, Control Unit), 입출력, 메모리가 통합된 하드웨어 중심의 단순 제어 장치로, serial ports, I2C, ADC, LCD 등 단순한 주변장치를 포함한다. 임베디드 시스템은 MCU에 임베디드 OS가 추가되어 네트워크, USB, 오디오, 비디오 코덱 등 복잡한 주변장치를 제어하며, IEEE 정의에 따르면 기계·장비·공장 등 대규모 시스템을 제어·모니터링하는 장치이다.

### Q2. 운영체제 설명 및 특징

운영체제는 응용 프로그램이나 사용자에게 컴퓨터 자원에 대한 인터페이스를 제공하는 시스템 소프트웨어로, 자원 관리/보호, 하드웨어 인터페이스 제공, 사용자 인터페이스 제공의 역할을 한다. 목표는 편리성(사용자 친화적 환경 및 프로그램 실행 플랫폼 제공), 효율성(CPU·메모리·디스크·I/O 자원의 활용도 최대화), 안정성(허가 받지 않은 접근으로부터 보호), 확장성(다양한 HW 추가/접근을 중재)이다.

### Q3. 프로세스와 스레드의 차이와 공통점

프로세스는 실행 중인 프로그램으로 독립된 메모리 주소 공간을 가지며 메모리 분리(memory separation)가 적용되고, 스레드는 프로세스 안에서 실행되는 줄기로 기본적으로 프로세스의 메모리를 모두 공유한다. 차이점은 프로세스는 부모/자식 상속관계가 있지만 스레드는 없고, 스레드는 생성 시 필요한 리소스/오버헤드가 프로세스보다 적다. 공통점은 둘 다 코드를 실행하는 단위이며 프로세스 내에서 병렬적으로 수행된다.

### Q4. 뮤텍스(Mutex)

뮤텍스는 상호 배제(mutual exclusion)의 약자로, 임계 영역(Critical Section)에 한 개의 스레드만 접근을 허용하여 Race condition을 방지하는 동기화 메커니즘이다. 임계 영역 진입 전에 lock, 이후에 unlock을 수행하여 한 스레드가 공유 변수 접근을 마칠 때까지 다른 스레드의 접근을 차단한다.

### Q5. 멀티태스킹과 오버헤드

멀티태스킹은 CPU가 빠르게 작업을 바꾸면서 실행하여 여러 작업이 동시에 처리되는 것처럼 보이게 하는 기술이며, 이를 위해 문맥 교환(Context Switching)으로 실행 중인 프로세스의 상태를 PCB에 저장하고 다음 프로세스의 상태를 PCB에서 복원한다. 오버헤드는 문맥 교환 과정 동안 CPU가 아무 일도 하지 않는 시간으로, 잦아지면 성능이 떨어질 수 있다.

### Q6. 메모리 계층(물리적)

메모리 계층은 응답 속도(Latency)와 크기(Capacity) 사이의 트레이드오프가 존재하며, 위에서부터 레지스터(가장 빠르고 가장 작음, 프로세서 내부 연산용), 캐시(CPU 옆 중간 버퍼, Cache Hit/Miss와 시간적·공간적 국소성 활용), DRAM(휘발성 메모리, 동작 중 데이터 저장), 디스크(비휘발성, 가장 느리고 가장 큼) 순서이다. 사용 확률이 높은 데이터를 더 빠른 메모리에 보관하여 접근 성능을 높이는 것이 핵심 원리이다.

### Q7. Memory-mapped I/O vs I/O Mapped I/O

I/O Mapped I/O는 메모리 영역과 I/O 영역이 확실하게 구분되어 각각 다른 명령어를 사용하며(인텔 x86), 메모리 영역을 온전하게 사용할 수 있지만 하드웨어 구성이 복잡하다. Memory-mapped I/O는 메모리 영역의 일부를 I/O 장치 주소로 사용하여 메모리와 I/O 접근 명령이 같으므로(ARM, MIPS) 프로세서가 단순하고 저렴하지만, 컴파일러 최적화 문제(volatile 키워드로 방지)와 캐시 메모리 문제(노캐시 영역 설정으로 해결)에 주의해야 한다.

### Q8. DMA

DMA(Direct Memory Access)는 메모리 장치와 입출력 장치가 CPU 개입 없이 직접 데이터를 송수신하는 방식으로, PIO 방식에서 프로세서가 항상 데이터를 중계해야 하는 문제를 해결한다. 사용자 프로그램이 통신 라이브러리를 통해 전송을 요청하면, 커널 드라이버가 가상 주소를 물리 페이지로 변환하고 메모리를 고정(pinning)한 뒤 DMA 엔진을 프로그래밍하여 전송을 수행한다.

### Q9. 리눅스 커널

리눅스 커널은 운영체제의 핵심 프로그램으로 컴퓨터의 물리적/추상화 자원(디바이스, 프로세스, 메모리 등)을 관리한다. 아키텍처는 User space(Application Code + C Library)와 Kernel space(System Interface → Generic Services → Device Drivers → Hardware)로 분리되며, 시스템 호출 인터페이스를 통해 상호작용하여 커널의 중요 자원에 일반 사용자가 접근하지 못하도록 보호한다.

## Part 2. 머신러닝 핵심개념 1 (데이터 전처리, 차원축소, 분류)

### Q1. 데이터 전처리의 정의와 주요 기법 6가지를 서술하시오.

데이터 전처리란 데이터의 품질을 올리는 과정이다. 주요 기법으로는 데이터 실수화(Data Vectorization), 데이터 정제(Data Cleaning), 데이터 통합(Data Integration), 데이터 축소(Data Reduction), 데이터 변환(Data Transformation), 데이터 균형(Data Balancing)이 있다.

### Q2. 표준화(Standardization)와 정규화(Normalization)의 수식과 차이점을 설명하시오.

표준화는 x_std = (x - mean(x)) / sd(x)로 평균 0, 표준편차 1로 변환하며, 정규화는 x_nor = (x - min(x)) / (max(x) - min(x))로 0~1 범위로 변환한다. 두 방법 중 어느 쪽이 항상 더 유용하다고 일반화할 수는 없으며, 데이터 분포와 사용하는 알고리즘에 따라 적절한 방법이 달라진다. 예를 들어 거리 기반 알고리즘에서는 정규화가, 평균과 분산의 스케일이 중요한 선형 모델이나 SVM 계열에서는 표준화가 자주 사용된다.

### Q3. One-hot encoding의 개념과 희소행렬(Sparse Matrix) 문제를 설명하시오.

One-hot encoding은 범주형 자료를 각 범주별 이진(0/1) 벡터로 변환하는 실수화 방법이다. 이 과정에서 대부분의 값이 0인 희소행렬이 생성되어 메모리 낭비와 연산 시간 증가 문제가 발생한다. 이를 해결하기 위해 COO 또는 CSR(Compressed Sparse Row) 표현식을 사용하여 메모리를 절약할 수 있다.

### Q4. TF-IDF의 개념과 단순 출현 횟수 기반 실수화 대비 장점을 서술하시오.

TF-IDF(Term Frequency-Inverse Document Frequency)는 단어의 출현 빈도에 가중치를 재계산하여, 높은 빈도의 단어에는 낮은 가중치를, 낮은 빈도의 단어에는 높은 가중치를 부여하는 기법이다. 단순 출현 횟수는 "the", "a" 같은 관사처럼 자주 등장하지만 분석에 의미 없는 단어도 높은 값을 갖는 문제가 있는데, TF-IDF는 이런 단어의 중요도를 낮춰 더 의미 있는 특성 표현이 가능하다.

### Q5. 과소표집(Undersampling)과 과대표집(Oversampling)의 개념과 차이를 비교하시오.

과소표집은 다수 클래스의 표본을 임의로 제거하여 클래스 비율을 맞추는 방법이고, 과대표집은 소수 클래스의 표본을 복제하여 학습 데이터에 추가하는 방법이다. 일반적으로 과대표집이 통계적으로 더 유용하며, 대표적인 과대표집 방법으로 SMOTE와 ADASYN이 있다.

### Q6. 차원의 저주(Curse of Dimensionality)의 개념과 이로 인해 발생하는 문제를 설명하시오.

차원의 저주란 차원이 증가할수록 동일 정보량을 표현하기 위해 필요한 데이터의 수가 지수적으로 증가하는 현상이다. 차원이 높아지면 데이터에 포함된 노이즈 비율이 높아지고, 모델 학습과 추론의 계산 복잡도가 증가하며, 개별 차원 내 학습할 데이터가 희박(sparse)해져 모델 성능이 저하된다.

### Q7. PCA(주성분 분석)의 목적과 주성분 개수 선정 방법 2가지를 서술하시오.

PCA는 데이터를 사영(projection)한 후 분산(variance)을 최대한 보존하는 기저를 찾아 차원을 줄이는 비지도 학습 방법이다. 주성분 개수 선정 방법으로는 첫째, 고유값 감소율이 유의미하게 낮아지는 Elbow Point를 선택하는 방법과, 둘째, 일정 수준 이상(보통 70~80%)의 누적 분산비를 보존하는 최소 주성분 수를 선택하는 방법이 있다.

### Q8. PCA와 LDA의 차원 축소 방식의 차이점을 비교하시오.

PCA는 비지도 학습 기반으로 데이터의 분산을 최대화하는 방향으로 차원을 축소하며, 클래스 레이블 정보를 사용하지 않는다. 반면 LDA는 지도 학습 기반으로 클래스 간 평균의 차이를 극대화하고 클래스 내 분산을 최소화하는 방향으로 차원을 축소한다. 따라서 PCA는 데이터 표현(signal representation)에, LDA는 분류(classification)에 더 적합하다.

### Q9. KNN의 개념과 하이퍼파라미터 K값에 따른 과적합/과소적합 문제를 설명하시오.

KNN은 새로운 데이터에 대해 주변 k개의 가장 가까운 학습 데이터의 클래스를 투표하여 분류하는 사례중심학습(instance-based learning) 방법이다. K가 작으면 데이터의 지역적 특성을 지나치게 반영하여 과적합(overfitting)이 발생하고, K가 매우 크면 모델이 과하게 정규화되어 과소적합(underfitting)이 발생한다. 최적의 K는 데이터마다 다르기 때문에 보통 교차검증(cross validation) 기반의 Grid Search 등 하이퍼파라미터 탐색 방법으로 선택한다.

### Q10. LDA(선형 판별 분석)의 가정 2가지와 이 가정이 위반될 때의 대안을 서술하시오.

LDA의 가정은 첫째, 각 클래스 집단이 정규분포(normal distribution) 형태의 확률분포를 가지며, 둘째, 각 클래스 집단이 비슷한 형태의 공분산(covariance) 구조를 가진다는 것이다. 만약 클래스별 공분산 구조가 서로 다른 경우에는 이차판별분석법(QDA)을 적용하여 해결할 수 있으며, QDA는 비선형 결정 경계를 형성할 수 있다는 장점이 있다.

### Q11. (수식) PCA의 수학적 원리를 Step별로 설명하고, 각 단계의 수식을 서술하시오.

1. **Step 1: 데이터 센터링 (Data Centering)**
   각 특성(변수)에서 해당 특성의 평균을 빼서 데이터의 평균을 0으로 변경한다. 이는 이후 공분산 행렬 계산의 기초가 된다.

2. **Step 2: 최적화 문제 정의**
   센터링된 데이터 `X`를 기저 벡터 `w`에 사영(projection)하면, 사영 후 분산 `V`는 다음과 같다.

   ```text
   V = (1/n)(w^T X)(w^T X)^T = (1/n)w^T XX^T w = w^T Sw
   ```

   여기서 `S = (1/n)XX^T` 는 `X`의 공분산 행렬(covariance matrix)이다. PCA의 목적은 사영 후 분산 `V`를 최대화하는 것이므로, 최적화 문제는 다음과 같이 정의된다.

   ```text
   max w^T Sw, s.t. w^T w = 1
   ```

   `w^T w = 1` 제약 조건은 기저 벡터의 크기를 1로 고정하여 분산 최대화가 방향에 의해서만 결정되도록 한다.

3. **Step 3: 라그랑지 멀티플라이어를 통한 풀이**
   제약 조건부 최적화를 위해 라그랑지안 함수를 구성한다.

   ```text
   L = w^T Sw − λ(w^T w − 1)
   ```

   `L`을 `w`에 대해 편미분하여 0으로 놓으면 다음과 같다.

   ```text
   ∂L/∂w = 0 → Sw − λw = 0 → (S − λI)w = 0
   ```

   이는 고유값 분해(eigenvalue decomposition) 문제로, `w`는 공분산 행렬 `S`의 고유벡터(eigenvector), `λ`는 대응하는 고유값(eigenvalue)이 된다.

4. **Step 4: 주축 정렬**
   구한 고유값을 크기 순서대로 정렬하고, 가장 큰 고유값에 대응하는 고유벡터를 첫 번째 주성분으로 선택한다. `w₁`에 사영된 데이터의 분산은 해당 고유값 `λ₁`과 같다.

   ```text
   v = w₁^T Sw₁ = w₁^T (λ₁w₁) = λ₁ w₁^T w₁ = λ₁
   ```

5. **Step 5: 데이터 변환**
   원 데이터 `X`에 선택된 고유벡터 `W₁`을 곱하여 주성분 `Z₁`을 얻는다.

   ```text
   Z₁ = W₁^T X
   ```

6. **Step 6: 원데이터 복원 (Reconstruction)**
   축소된 데이터를 다시 원래 차원으로 복원할 수 있다.

   ```text
   X → w^T X (Projection) → ww^T X (Reconstruction)
   ```

   복원된 데이터 `X'`는 원본과 완전히 동일하지는 않으며, 선택하지 않은 주성분의 정보만큼 손실이 발생한다. 첫 번째 주성분만으로 설명 가능한 데이터 비율은 다음과 같다.

   ```text
   λ₁ / (λ₁ + λ₂ + ... + λₙ)
   ```

## Part 3. 머신러닝 핵심개념 2 (SVM, 군집화, 의사결정나무, 앙상블)

### Q1. SVM에서 Hard Margin SVM과 Soft Margin SVM의 차이를 목적 함수와 함께 서술하시오.

Hard Margin SVM은 선형 분리 가능한 데이터에 적용되며 목적 함수는 min (1/2)‖w‖₂², 제약조건 yᵢ(wᵀxᵢ+b) ≥ 1로 오분류를 허용하지 않는다. Soft Margin SVM은 선형 분리 불가능한 경우 슬랙 변수 ξᵢ를 도입하여 목적 함수를 min (1/2)‖w‖₂² + C·Σξᵢ로 확장하고, 제약조건을 yᵢ(wᵀxᵢ+b) ≥ 1−ξᵢ, ξᵢ ≥ 0으로 변경하여 에러를 허용한다. C가 크면 에러를 적게 허용(마진 작아짐, Overfitting), C가 작으면 에러를 많이 허용(마진 커짐, Underfitting)한다.

### Q2. 커널 트릭(Kernel Trick)의 개념과 사용 이유를 서술하고, 대표적인 커널 함수 3가지를 쓰시오.

커널 트릭은 원래 공간에서 선형 분리가 불가능한 데이터를 Feature Map Φ로 고차원 공간에 매핑한 뒤, Φ(X)와 Φ(Y)를 명시적으로 각각 계산하지 않고 커널 함수 K(X,Y) = ⟨Φ(X),Φ(Y)⟩를 통해 암묵적으로 내적을 바로 계산하여 연산 효율을 높이는 기법이다. 대표적 커널 함수는 Polynomial K(x₁,x₂) = (a⟨x₁,x₂⟩+b)^d, Sigmoid K(x₁,x₂) = tanh(a⟨x₁,x₂⟩+b), Gaussian(RBF) K(x₁,x₂) = exp(−‖x₁−x₂‖₂²/2σ²) 이다.

### Q3. (수식) SVM의 마진(Margin)이 2/‖W‖₂ 임을 유도하시오.

- 결정 경계와 양쪽 초평면은 다음과 같다.

  ```text
  결정 경계: WᵀX + b = 0
  양의 초평면: WᵀX + b = +1
  음의 초평면: WᵀX + b = −1
  ```

- 서로 다른 클래스의 서포트 벡터 관계를 다음과 같이 둔다.

  ```text
  X⁺ = X⁻ + λW
  ```

- 이를 `WᵀX⁺ + b = 1`에 대입하면,

  ```text
  Wᵀ(X⁻ + λW) + b = 1
  → (WᵀX⁻ + b) + λWᵀW = 1
  → −1 + λWᵀW = 1
  → λ = 2 / (WᵀW)
  ```

- 따라서 마진은 다음과 같이 유도된다.

  ```text
  Margin = ‖X⁺ − X⁻‖₂
         = ‖λW‖₂
         = λ√(WᵀW)
         = (2 / WᵀW) · √(WᵀW)
         = 2 / √(WᵀW)
         = 2 / ‖W‖₂
  ```

- 결론: **SVM의 마진은 `2/‖W‖₂`이다.**

### Q4. K겹 교차 검증(K-fold Cross Validation)의 절차와, 홀드아웃 방법 대비 장점을 서술하시오.

K겹 교차 검증은 중복 없이 훈련 데이터를 K개의 겹(fold)으로 랜덤하게 나눈 뒤, K−1겹으로 모델을 훈련하고 나머지 1겹으로 성능을 평가하는 과정을 K번 반복하여 평균 성능을 계산하는 방법이다. 홀드아웃 방법 대비 데이터 분할에 덜 예민한 성능 평가가 가능하고, 모든 샘플이 검증에 딱 한번씩 사용되어 데이터 활용 효율이 높다. 일반적으로 K=10을 추천하며, K가 클수록 실행 시간이 길어지므로 큰 데이터에는 작은 K를 선택해도 된다.

### Q5. 과대적합(Overfitting)과 과소적합(Underfitting)의 개념과 각각의 해결 방법을 서술하시오.

과대적합은 모델이 학습 데이터에 너무 잘 맞지만 일반화(generalization) 성능이 떨어지는 현상으로, 해결 방법은 학습 데이터 추가 수집, 규제(regularization) 값 높이기, 학습 데이터 잡음 줄이기(오류 수정/이상치 제거)이다. 과소적합은 모델이 데이터에 내재된 구조를 학습하지 못하는 현상으로, 해결 방법은 파라미터가 더 많은 모델 선택, 규제 값 줄이기, 과적합 이전까지 충분히 학습하기이다.

### Q6. 의사결정나무(Decision Tree)의 개념과 장단점을 서술하시오.

의사결정나무는 학습 데이터에 내재된 패턴을 분석하여 If-then 규칙의 나무 구조로 표현하고, 새로운 데이터를 예측·분류하는 지도 학습 모델이다. 장점은 나무 구조로 모델 해석이 용이하고 의사결정 과정 설명이 가능하며, 데이터의 통계적 가정이 불필요하고, 중요한 변수 선택에 유용하다는 점이다. 단점은 좋은 모형을 만들기 위해 많은 데이터와 시간이 필요하고, 데이터 변화에 민감하며(domain gap 작아야 함), 선형 구조형 데이터 예측 시 복잡해진다는 점이다.

### Q7. 의사결정나무의 재귀적 분할 알고리즘 CART, C4.5, CHAID의 불순도 지표와 분리 방식의 차이를 서술하시오.

CART는 Gini index를 불순도 지표로 사용하며 이진 분할(Binary split)을 수행하고, 완전 모형 생성 후 가지치기(Pruning)를 한다. C4.5는 엔트로피(Entropy) 기반 정보 이익(Information Gain)과 정보 이익 비율(Information Gain Ratio)을 사용하며 다중 분할(Multi-way split)을 수행한다. CHAID는 카이제곱 통계량(Chi-square statistic)을 사용하며 다중 분할을 수행하되, 완전 모형을 생성하지 않고 최적 모형을 직접 개발하므로 별도 가지치기가 없다.

### Q8. (수식) CART에서 사용하는 Gini Index의 수식을 쓰고, 이 값이 의미하는 바를 서술하시오.

G.I.(A) = Σ(i=1→d) [ Rᵢ · (1 − Σ(k=1→m) pᵢₖ²) ]

여기서 Rᵢ는 i번째 자식 노드의 데이터 비율, pᵢₖ는 i번째 노드에서 클래스 k의 비율이다. 1 − Σpᵢₖ² = Σpᵢₖ(1−pᵢₖ)으로 변환할 수 있으며, 이는 해당 노드의 클래스 분포를 바탕으로 무작위 예측했을 때 예상되는 오분류 확률의 기댓값을 의미한다. Gini index가 낮을수록 노드의 순도가 높아 분류가 잘 된 것이다.

### Q9. 앙상블 학습에서 배깅(Bagging)과 부스팅(Boosting)의 차이를 서술하시오.

배깅은 알고리즘 수행마다 부트스트랩(Bootstrap, 복원 추출)으로 서로 다른 학습 데이터를 샘플링하여 여러 모델을 병렬적으로 구축한 뒤 투표/평균으로 결합하는 방식이다. 부스팅은 이전 모델에서 잘못 분류된 데이터에 가중치를 부여하여 다음 모델이 이를 더 잘 학습하도록 순차적으로 weak learner를 조합해 strong learner를 만드는 방식이다. 배깅의 대표 모델은 Random Forest이고, 부스팅의 대표 모델은 AdaBoost이다.

### Q10. 랜덤 포레스트(Random Forest)의 분석 절차를 서술하시오.

먼저 크기 n, 특성 변수 d개인 원본 학습 데이터에서 Bootstrap(복원 추출)을 통해 n개의 새로운 학습 데이터를 생성한다. 그 다음 d개의 특성 변수 중 임의로 k개(일반적으로 분류 문제에서는 k≈√d)의 특성 변수를 뽑아 의사결정나무를 구성한다. 이 절차를 M번 반복하여 M개의 트리를 만들고, 분류 문제는 투표, 예측 문제는 평균화로 최종 결과를 결합한다. 일반적으로 M이 증가할수록 분산이 줄고 성능이 안정되지만, 어느 시점 이후에는 성능 향상이 거의 포화된다.

## Part 4. 딥러닝 기초 1

### Q1. Traditional AI 접근법의 한계를 Polanyi's Paradox와 연관지어 설명하시오.

Traditional AI는 인간 전문가의 지식을 IF-THEN 규칙으로 명시적으로 프로그래밍하는 방식이다. 그러나 Polanyi's Paradox에 따르면, 인간은 자신이 수행하는 많은 작업을 정확히 어떻게 하는지 설명하기 어렵다. 따라서 모든 상황과 예외를 규칙으로 작성할 수 없어 새로운 상황에 일반화하지 못하고, 소수의 영역에서만 성공할 수 있었다.

### Q2. Machine Learning과 Deep Learning의 핵심적인 차이점을 데이터 표현(representation) 관점에서 설명하시오.

Machine Learning은 비정형 데이터를 다루기 위해 인간이 수동으로 특징(feature)을 설계하여 정형 데이터로 변환한 후 학습하는 반면, Deep Learning은 원시 비정형 데이터로부터 자동으로 효과적인 표현을 학습한다. 이로 인해 Deep Learning은 수동 전처리 없이도 이미지, 텍스트, 오디오 등 비정형 데이터를 직접 처리할 수 있어 "human bottleneck" 문제를 해결한다.

### Q3. (수식) 신경망에서 activation function이 반드시 비선형(non-affine)이어야 하는 이유를 수식으로 설명하시오.

- 만약 활성화 함수 `σ`가 affine 변환이라면,

  ```text
  σ(x) = Wx + b
  ```

- 각 층의 출력은 다음과 같이 축약된다.

  ```text
  x⁽ˡ⁾ = σ(w⁽ˡ⁾x⁽ˡ⁻¹⁾ + b⁽ˡ⁾)
       = W(w⁽ˡ⁾x⁽ˡ⁻¹⁾ + b⁽ˡ⁾) + b
       = W'x⁽ˡ⁻¹⁾ + b'
  ```

- 이를 여러 층에 반복 적용하면,

  ```text
  x⁽ˡ⁾ = W'x⁽ˡ⁻¹⁾ + b'
       = W''x⁽ˡ⁻²⁾ + b''
       = ⋯
       = W'''x⁽⁰⁾ + b'''
  ```

- 즉, 여러 층을 쌓아도 전체 네트워크가 결국 하나의 affine 변환으로 축약된다. 따라서 다층 구조의 표현력이 생기려면 **비선형 활성화 함수가 필수적이다.**

### Q4. Sigmoid, Tanh, ReLU 활성화 함수의 장단점을 각각 비교하시오.

Sigmoid는 출력이 (0,1) 범위로 확률 예측에 적합하나, 입력의 절대값이 클 때 gradient가 거의 0이 되는 vanishing gradient 문제가 있다. Tanh는 출력이 zero-centered(-1,1)로 학습이 용이하고 gradient가 더 가파르지만, 역시 vanishing gradient 문제를 가진다. ReLU는 max(0,x)로 계산이 효율적이고 수렴 속도가 빠르지만, 음수 입력에서 gradient가 0이 되어 뉴런이 영구적으로 비활성화되는 dying ReLU 문제가 존재한다.

### Q5. 신경망에서 weight, bias, neuron, layer, activation function 각 용어의 의미를 네트워크 구조와 연관지어 설명하시오.

Weight는 각 입력 노드의 값에 곱해지는 계수(coefficient)이고, bias는 가중합에 더해지는 절편(intercept)이다. Neuron은 가중합(+)과 활성화 함수 f(x)를 하나로 묶은 연산 단위이며, layer는 이러한 뉴런들의 수직적 묶음이다. Activation function은 각 뉴런의 가중합 출력에 적용되는 비선형 함수로, 네트워크에 비선형성을 부여한다.

### Q6. (수식) Perceptron의 수렴 정리(Convergence Theorem)의 두 가지 가정과 수렴 상한을 수식으로 서술하시오.

- **가정 1:** 모든 데이터 `xₙ`이 반경 `R`인 구 안에 존재한다.

  ```text
  ∃R > 0, ∀n, ‖xₙ‖ ≤ R
  ```

- **가정 2:** 단위 벡터 `w*`가 존재하여 마진 `γ`로 두 클래스를 분리한다.

  ```text
  ∃w*, ‖w*‖ = 1, ∃γ > 0, ∀n, yₙ(xₙ · w*) ≥ γ/2
  ```

- 수렴 증명의 핵심은 업데이트 규칙

  ```text
  w^(k+1) = w^k + yₙₖxₙₖ
  ```

  에 대해 아래의 상한과 하한을 얻는 것이다.

- **하한**

  ```text
  w^(k+1) · w* ≥ (k+1)γ/2
  → ‖w^(k+1)‖ · ‖w*‖ ≥ (k+1)γ/2
  → ‖w^(k+1)‖ ≥ (k+1)γ/2
  ```

- **상한**

  ```text
  ‖w^(k+1)‖² ≤ ‖w^k‖² + R² ≤ (k+1)R²
  ```

- 두 부등식을 결합하면,

  ```text
  ((k+1)²γ²)/4 ≤ ‖w^(k+1)‖² ≤ (k+1)R²
  → (k+1)γ²/4 ≤ R²
  → k ≤ 4R²/γ²
  ```

- 따라서 최대 `⌊4R²/γ²⌋`번의 업데이트 후 수렴이 보장된다.

### Q7. Gradient Descent 알고리즘의 원리와, 실제 딥러닝에서 mini-batch를 사용하는 이유를 설명하시오.

Gradient Descent는 손실 함수의 기울기(gradient) 반대 방향으로 파라미터를 반복 업데이트(θ ← θ - α∇ℒ(θ))하여 손실을 최소화하는 알고리즘이다. 전체 데이터셋에 대한 gradient 계산은 데이터가 클 때(예: ImageNet 150만 장) 매우 비용이 크므로, B개의 샘플로 구성된 mini-batch로 gradient를 근사하여 한 번에 더 빠르게 업데이트한다. 배치 크기가 1이면 순수 SGD(Stochastic Gradient Descent)라고 하고, 실제 딥러닝에서는 보통 mini-batch SGD를 사용한다. 이 방식은 경로가 다소 불안정할 수 있지만 같은 시간에 더 많은 업데이트가 가능하다.

### Q8. Backpropagation에서 chain rule을 활용하여 gradient를 효율적으로 계산하는 원리를 설명하시오.

신경망은 함수의 합성(composition)이므로, 각 파라미터에 대한 손실의 gradient는 chain rule로 각 층의 Jacobian을 곱하여 계산한다. 이때 오른쪽(출력 쪽)부터 벡터-행렬 곱 형태(δ를 전파)로 순차적으로 계산하면, n×n 행렬 곱셈(O(n³)) 대신 행렬-벡터 곱(O(n²))으로 효율적으로 수행할 수 있다. Forward pass에서 각 층의 중간값을 저장한 후, backward pass에서 출력층부터 입력층 방향으로 δ를 전파하며 각 파라미터의 gradient를 계산한다.

### Q9. Newton's method가 gradient descent보다 빠르게 수렴할 수 있음에도 딥러닝에서 사용되지 않는 이유를 설명하시오.

Newton's method는 2차 Taylor 전개를 이용하여 θ\* ← θ₀ − (∇²ℒ(θ₀))⁻¹∇ℒ(θ₀)로 최적점을 직접 근사하므로 수렴이 빠르다. 그러나 Hessian 행렬 ∇²ℒ는 n×n 크기로, n이 수백만~수억인 딥러닝에서는 저장 및 역행렬 계산의 시간 복잡도가 O(n³)이어서 실용적이지 않다. 따라서 Hessian 없이 gradient만으로 가속하는 Momentum, Adam 등의 1차 최적화 방법을 대신 사용한다.

### Q10. Momentum 기법의 직관적 원리와 업데이트 규칙을 기존 gradient descent와 비교하여 설명하시오.

기존 GD는 현재 gradient만 사용하여 θₖ₊₁ = θₖ - α∇θℒ(θₖ)로 업데이트한다. 반면 Momentum은 이전 업데이트 방향을 함께 누적하기 위해 속도 변수 v를 두고 다음과 같이 계산한다.

```text
vₖ = μvₖ₋₁ - α∇θℒ(θₖ)
θₖ₊₁ = θₖ + vₖ
```

여기서 μ는 모멘텀 계수이다. 연속된 gradient가 같은 방향이면 그 방향으로 가속되고, 반대 방향이면 일부 상쇄되어 진동이 감소한다. 이를 통해 loss surface에서 좁고 긴 계곡(elongated valley) 형태의 지형을 더 효율적으로 탐색하여 수렴 속도를 높인다.
