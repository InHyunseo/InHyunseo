## Part 1. 임베디드 시스템 개요

### Q1. MCU와 임베디드 시스템의 차이

MCU(마이크로컨트롤러)는 마이크로프로세서(ALU, Register, Control Unit), 입출력, 메모리가 통합된 하드웨어 중심의 단순 제어 장치로, serial ports, I2C, ADC, LCD 등 단순한 주변장치를 포함한다. 임베디드 시스템은 MCU에 임베디드 OS가 추가되어 네트워크, USB, 오디오, 비디오 코덱 등 복잡한 주변장치를 제어하며, IEEE 정의에 따르면 기계·장비·공장 등 대규모 시스템을 제어·모니터링하는 장치이다.

### Q2. 운영체제 설명 및 특징

운영체제는 응용 프로그램이나 사용자에게 컴퓨터 자원에 대한 인터페이스를 제공하는 시스템 소프트웨어로, 자원 관리/보호, 하드웨어 인터페이스 제공, 사용자 인터페이스 제공의 역할을 한다. 목표는 편리성(사용자 친화적 환경 및 프로그램 실행 플랫폼 제공), 효율성(CPU·메모리·디스크·I/O 자원의 활용도 최대화), 안정성(허가 받지 않은 접근으로부터 보호), 확장성(다양한 HW 추가/접근을 중재)이다.

### Q3. 프로세스와 스레드의 차이와 공통점

프로세스는 실행 중인 프로그램으로 독립된 메모리 주소 공간을 가지며 메모리 분리(memory separation)가 적용되고, 스레드는 프로세스 안에서 실행되는 줄기로 기본적으로 프로세스의 메모리를 모두 공유한다. 차이점은 프로세스는 부모/자식 상속관계가 있지만 스레드는 없고, 스레드는 생성 시 필요한 리소스/오버헤드가 프로세스보다 적다. 공통점은 둘 다 코드를 실행하는 단위이며 프로세스 내에서 병렬적으로 수행된다.

### Q4. 뮤텍스(Mutex)

뮤텍스는 상호 배제(mutual exclusion)의 약자로, 임계 영역(Critical Section)에 한 개의 스레드만 접근을 허용하여 Race condition을 방지하는 동기화 메커니즘이다. 임계 영역 진입 전에 lock, 이후에 unlock을 수행하여 한 스레드가 공유 변수 접근을 마칠 때까지 다른 스레드의 접근을 차단한다.

### Q5. 멀티태스킹과 오버헤드

멀티태스킹은 CPU가 빠르게 작업을 바꾸면서 실행하여 여러 작업이 동시에 처리되는 것처럼 보이게 하는 기술이며, 이를 위해 문맥 교환(Context Switching)으로 실행 중인 프로세스의 상태를 PCB에 저장하고 다음 프로세스의 상태를 PCB에서 복원한다. 오버헤드는 문맥 교환 과정 동안 CPU가 아무 일도 하지 않는 시간으로, 잦아지면 성능이 떨어질 수 있다.

### Q6. 메모리 계층(물리적)

메모리 계층은 응답 속도(Latency)와 크기(Capacity) 사이의 트레이드오프가 존재하며, 위에서부터 레지스터(가장 빠르고 가장 작음, 프로세서 내부 연산용), 캐시(CPU 옆 중간 버퍼, Cache Hit/Miss와 시간적·공간적 국소성 활용), DRAM(휘발성 메모리, 동작 중 데이터 저장), 디스크(비휘발성, 가장 느리고 가장 큼) 순서이다. 사용 확률이 높은 데이터를 더 빠른 메모리에 보관하여 접근 성능을 높이는 것이 핵심 원리이다.

### Q7. Memory-mapped I/O vs I/O Mapped I/O

I/O Mapped I/O는 메모리 영역과 I/O 영역이 확실하게 구분되어 각각 다른 명령어를 사용하며(인텔 x86), 메모리 영역을 온전하게 사용할 수 있지만 하드웨어 구성이 복잡하다. Memory-mapped I/O는 메모리 영역의 일부를 I/O 장치 주소로 사용하여 메모리와 I/O 접근 명령이 같으므로(ARM, MIPS) 프로세서가 단순하고 저렴하지만, 컴파일러 최적화 문제(volatile 키워드로 방지)와 캐시 메모리 문제(노캐시 영역 설정으로 해결)에 주의해야 한다.

### Q8. DMA

DMA(Direct Memory Access)는 메모리 장치와 입출력 장치가 CPU 개입 없이 직접 데이터를 송수신하는 방식으로, PIO 방식에서 프로세서가 항상 데이터를 중계해야 하는 문제를 해결한다. 사용자 프로그램이 통신 라이브러리를 통해 전송을 요청하면, 커널 드라이버가 가상 주소를 물리 페이지로 변환하고 메모리를 고정(pinning)한 뒤 DMA 엔진을 프로그래밍하여 전송을 수행한다.

### Q9. 리눅스 커널

리눅스 커널은 운영체제의 핵심 프로그램으로 컴퓨터의 물리적/추상화 자원(디바이스, 프로세스, 메모리 등)을 관리한다. 아키텍처는 User space(Application Code + C Library)와 Kernel space(System Interface → Generic Services → Device Drivers → Hardware)로 분리되며, 시스템 호출 인터페이스를 통해 상호작용하여 커널의 중요 자원에 일반 사용자가 접근하지 못하도록 보호한다.

## Part 2. 머신러닝 핵심개념 1 (데이터 전처리, 차원축소, 분류)

### Q1. 데이터 전처리의 정의와 주요 기법 6가지를 서술하시오.

데이터 전처리란 데이터의 품질을 올리는 과정이다. 주요 기법으로는 데이터 실수화(Data Vectorization), 데이터 정제(Data Cleaning), 데이터 통합(Data Integration), 데이터 축소(Data Reduction), 데이터 변환(Data Transformation), 데이터 균형(Data Balancing)이 있다.

### Q2. 표준화(Standardization)와 정규화(Normalization)의 수식과 차이점을 설명하시오.

표준화는 $x_{\mathrm{std}} = \frac{x - \mathrm{mean}(x)}{\mathrm{sd}(x)}$로 평균 0, 표준편차 1로 변환하며, 정규화는 $x_{\mathrm{nor}} = \frac{x - \min(x)}{\max(x) - \min(x)}$로 0~1 범위로 변환한다. 두 방법 중 어느 쪽이 항상 더 유용하다고 일반화할 수는 없으며, 데이터 분포와 사용하는 알고리즘에 따라 적절한 방법이 달라진다. 예를 들어 거리 기반 알고리즘에서는 정규화가, 평균과 분산의 스케일이 중요한 선형 모델이나 SVM 계열에서는 표준화가 자주 사용된다.

### Q3. One-hot encoding의 개념과 희소행렬(Sparse Matrix) 문제를 설명하시오.

One-hot encoding은 범주형 자료를 각 범주별 이진(0/1) 벡터로 변환하는 실수화 방법이다. 이 과정에서 대부분의 값이 0인 희소행렬이 생성되어 메모리 낭비와 연산 시간 증가 문제가 발생한다. 이를 해결하기 위해 COO 또는 CSR(Compressed Sparse Row) 표현식을 사용하여 메모리를 절약할 수 있다.

### Q4. TF-IDF의 개념과 단순 출현 횟수 기반 실수화 대비 장점을 서술하시오.

TF-IDF(Term Frequency-Inverse Document Frequency)는 단어의 출현 빈도에 가중치를 재계산하여, 높은 빈도의 단어에는 낮은 가중치를, 낮은 빈도의 단어에는 높은 가중치를 부여하는 기법이다. 단순 출현 횟수는 "the", "a" 같은 관사처럼 자주 등장하지만 분석에 의미 없는 단어도 높은 값을 갖는 문제가 있는데, TF-IDF는 이런 단어의 중요도를 낮춰 더 의미 있는 특성 표현이 가능하다.

### Q5. 과소표집(Undersampling)과 과대표집(Oversampling)의 개념과 차이를 비교하시오.

과소표집은 다수 클래스의 표본을 임의로 제거하여 클래스 비율을 맞추는 방법이고, 과대표집은 소수 클래스의 표본을 복제하여 학습 데이터에 추가하는 방법이다. 일반적으로 과대표집이 통계적으로 더 유용하며, 대표적인 과대표집 방법으로 SMOTE와 ADASYN이 있다.

### Q6. 차원의 저주(Curse of Dimensionality)의 개념과 이로 인해 발생하는 문제를 설명하시오.

차원의 저주란 차원이 증가할수록 동일 정보량을 표현하기 위해 필요한 데이터의 수가 지수적으로 증가하는 현상이다. 차원이 높아지면 데이터에 포함된 노이즈 비율이 높아지고, 모델 학습과 추론의 계산 복잡도가 증가하며, 개별 차원 내 학습할 데이터가 희박(sparse)해져 모델 성능이 저하된다.

### Q7. PCA(주성분 분석)의 목적과 주성분 개수 선정 방법 2가지를 서술하시오.

PCA는 데이터를 사영(projection)한 후 분산(variance)을 최대한 보존하는 기저를 찾아 차원을 줄이는 비지도 학습 방법이다. 주성분 개수 선정 방법으로는 첫째, 고유값 감소율이 유의미하게 낮아지는 Elbow Point를 선택하는 방법과, 둘째, 일정 수준 이상(보통 70~80%)의 누적 분산비를 보존하는 최소 주성분 수를 선택하는 방법이 있다.

### Q8. PCA와 LDA의 차원 축소 방식의 차이점을 비교하시오.

PCA는 비지도 학습 기반으로 데이터의 분산을 최대화하는 방향으로 차원을 축소하며, 클래스 레이블 정보를 사용하지 않는다. 반면 LDA는 지도 학습 기반으로 클래스 간 평균의 차이를 극대화하고 클래스 내 분산을 최소화하는 방향으로 차원을 축소한다. 따라서 PCA는 데이터 표현(signal representation)에, LDA는 분류(classification)에 더 적합하다.

### Q9. KNN의 개념과 하이퍼파라미터 K값에 따른 과적합/과소적합 문제를 설명하시오.

KNN은 새로운 데이터에 대해 주변 k개의 가장 가까운 학습 데이터의 클래스를 투표하여 분류하는 사례중심학습(instance-based learning) 방법이다. K가 작으면 데이터의 지역적 특성을 지나치게 반영하여 과적합(overfitting)이 발생하고, K가 매우 크면 모델이 과하게 정규화되어 과소적합(underfitting)이 발생한다. 최적의 K는 데이터마다 다르기 때문에 보통 교차검증(cross validation) 기반의 Grid Search 등 하이퍼파라미터 탐색 방법으로 선택한다.

### Q10. LDA(선형 판별 분석)의 가정 2가지와 이 가정이 위반될 때의 대안을 서술하시오.

LDA의 가정은 첫째, 각 클래스 집단이 정규분포(normal distribution) 형태의 확률분포를 가지며, 둘째, 각 클래스 집단이 비슷한 형태의 공분산(covariance) 구조를 가진다는 것이다. 만약 클래스별 공분산 구조가 서로 다른 경우에는 이차판별분석법(QDA)을 적용하여 해결할 수 있으며, QDA는 비선형 결정 경계를 형성할 수 있다는 장점이 있다.

### Q11. (수식) PCA의 수학적 원리를 Step별로 설명하고, 각 단계의 수식을 서술하시오.

1. **Step 1: 데이터 센터링 (Data Centering)**
   각 특성(변수)에서 해당 특성의 평균을 빼서 데이터의 평균을 0으로 변경한다. 이는 이후 공분산 행렬 계산의 기초가 된다.

2. **Step 2: 최적화 문제 정의**
   센터링된 데이터 `X`를 기저 벡터 `w`에 사영(projection)하면, 사영 후 분산 `V`는 다음과 같다.

   ```math
   V = \frac{1}{n}(w^T X)(w^T X)^T = \frac{1}{n}w^T XX^T w = w^T Sw
   ```

   여기서 $S = \frac{1}{n}XX^T$는 `X`의 공분산 행렬(covariance matrix)이다. PCA의 목적은 사영 후 분산 `V`를 최대화하는 것이므로, 최적화 문제는 다음과 같이 정의된다.

   ```math
   \max \; w^T S w,\ s.t.\ w^T w = 1
   ```

   $w^T w = 1$ 제약 조건은 기저 벡터의 크기를 1로 고정하여 분산 최대화가 방향에 의해서만 결정되도록 한다.

3. **Step 3: 라그랑지 멀티플라이어를 통한 풀이**
   제약 조건부 최적화를 위해 라그랑지안 함수를 구성한다.

   ```math
   L = w^T S w - \lambda (w^T w - 1)
   ```

   `L`을 `w`에 대해 편미분하여 0으로 놓으면 다음과 같다.

   ```math
   \frac{\partial L}{\partial w} = 0 \Rightarrow Sw - \lambda w = 0 \Rightarrow (S - \lambda I)w = 0
   ```

   이는 고유값 분해(eigenvalue decomposition) 문제로, $w$는 공분산 행렬 $S$의 고유벡터(eigenvector), $\lambda$는 대응하는 고유값(eigenvalue)이 된다.

4. **Step 4: 주축 정렬**
   구한 고유값을 크기 순서대로 정렬하고, 가장 큰 고유값에 대응하는 고유벡터를 첫 번째 주성분으로 선택한다. $w_1$에 사영된 데이터의 분산은 해당 고유값 $\lambda_1$과 같다.

   ```math
   v = w_1^T S w_1 = w_1^T (\lambda_1 w_1) = \lambda_1 w_1^T w_1 = \lambda_1
   ```

5. **Step 5: 데이터 변환**
   원 데이터 $X$에 선택된 고유벡터 $W_1$을 곱하여 주성분 $Z_1$을 얻는다.

   ```math
   Z_1 = W_1^T X
   ```

6. **Step 6: 원데이터 복원 (Reconstruction)**
   축소된 데이터를 다시 원래 차원으로 복원할 수 있다.

   ```math
   X \rightarrow w^T X \rightarrow ww^T X
   ```

   복원된 데이터 `X'`는 원본과 완전히 동일하지는 않으며, 선택하지 않은 주성분의 정보만큼 손실이 발생한다. 첫 번째 주성분만으로 설명 가능한 데이터 비율은 다음과 같다.

   ```math
   \frac{\lambda_1}{\lambda_1 + \lambda_2 + \cdots + \lambda_n}
   ```

### [개념] Q1. SVM의 정의와 학습 방향을 서술하시오.

SVM(Support Vector Machine)은 패턴인식과 자료 분석을 위한 지도 학습 모델로, 두 카테고리 중 어느 하나에 속한 데이터 집합을 바탕으로 새로운 데이터가 어느 카테고리에 속할지 판단하는 비-확률적 이진 선형 분류 모델이다. SVM의 학습 방향은 마진(Margin)의 최대화이며, 결정 경계(Hyperplane)가 주변 데이터와의 거리가 최대가 되어야 결정 경계 근처에 위치하는 새로운 데이터가 들어와도 강인한 분류가 가능하다.

### [개념] Q2. SVM에서 결정 경계(Hyperplane), 서포트 벡터(Support Vector), 마진(Margin)의 정의를 서술하시오.

결정 경계는 서로 다른 클래스를 완벽하게 분류하는 기준으로, 데이터 임베딩 공간보다 1차원 낮은 부분 공간이다(2차원→직선, 3차원→평면, 4차원 이상→초평면). 서포트 벡터는 결정 경계선에 가장 가까이에 있는 각 클래스의 데이터이다. 마진은 어떤 데이터도 포함하지 않는 영역으로, 서포트 벡터와 결정 경계 사이의 거리이다.

### [수식] Q3. SVM의 마진(Margin)이 $2/\|W\|_2$ 임을 유도하시오.

결정 경계와 양쪽 초평면은 다음과 같다.

```math
\begin{aligned}
\text{결정 경계: } & W^T X + b = 0 \\
\text{양의 초평면: } & W^T X + b = +1 \\
\text{음의 초평면: } & W^T X + b = -1
\end{aligned}
```

서로 다른 클래스의 서포트 벡터 관계를 $X^+ = X^- + \lambda W$로 두고, $W^T X^+ + b = 1$에 대입하면,

```math
\begin{aligned}
W^T(X^- + \lambda W) + b &= 1 \\
\Rightarrow (W^T X^- + b) + \lambda W^T W &= 1 \\
\Rightarrow -1 + \lambda W^T W &= 1 \\
\Rightarrow \lambda &= \frac{2}{W^T W}
\end{aligned}
```

따라서 마진은 다음과 같이 유도된다.

```math
\begin{aligned}
\mathrm{Margin}
&= \|X^+ - X^-\|_2 = \|\lambda W\|_2 = \lambda \sqrt{W^T W} \\
&= \left( \frac{2}{W^T W} \right)\sqrt{W^T W}
= \frac{2}{\sqrt{W^T W}}
= \frac{2}{\|W\|_2}
\end{aligned}
```

### [수식] Q4. Hard Margin SVM과 Soft Margin SVM의 목적 함수와 제약조건을 쓰고, 차이를 서술하시오.

Hard Margin SVM은 선형 분리 가능한 데이터에 적용되며 목적 함수와 제약조건은 다음과 같다.

```math
\begin{aligned}
\min_{w,b}\;& \frac{1}{2}\|w\|_2^2 \\
\text{subject to }\;& y_i(w^T x_i + b) \ge 1,\quad i = 1,2,\ldots,n
\end{aligned}
```

Soft Margin SVM은 선형 분리 불가능한 경우 슬랙 변수 $\xi_i$를 도입하여 에러를 허용한다.

```math
\begin{aligned}
\min_{w,b,\xi}\;& \frac{1}{2}\|w\|_2^2 + C \sum \xi_i \\
\text{subject to }\;& y_i(w^T x_i + b) \ge 1 - \xi_i,\quad \xi_i \ge 0,\quad i = 1,2,\ldots,n
\end{aligned}
```

C가 크면 학습 에러를 상대적으로 허용하지 않아 마진이 작아지고 과적합(Overfitting) 경향이 있으며, C가 작으면 에러를 많이 허용하여 마진이 커지고 과소적합(Underfitting) 경향이 있다.

### [수식] Q5. SVM의 라그랑지안 Primal Problem과 Dual Problem을 수식으로 쓰고, KKT 조건 중 Complementary Slackness가 SVM에서 갖는 의미를 설명하시오.

Hard Margin SVM의 Primal Problem은 제약식을 목적식에 포함시켜 다음과 같이 구성된다.

```math
\begin{aligned}
\max_{\alpha}\min_{w,b}\; \mathcal{L}(w,b,\alpha)
= \frac{1}{2}\|w\|_2^2 - \sum \alpha_i \bigl(y_i(w^T x_i + b) - 1\bigr) \\
\text{subject to }\; \alpha_i \ge 0
\end{aligned}
```

이를 Dual Problem으로 변환하면 다음과 같다.

```math
\begin{aligned}
\max_{\alpha}\;& \sum \alpha_i - \frac{1}{2}\sum\sum \alpha_i \alpha_j y_i y_j x_i^T x_j \\
\text{subject to }\;& \sum \alpha_i y_i = 0,\quad \alpha_i \ge 0 \quad (\text{Hard Margin}) \\
\text{subject to }\;& \sum \alpha_i y_i = 0,\quad 0 \le \alpha_i \le C \quad (\text{Soft Margin})
\end{aligned}
```

Complementary Slackness 조건 $\alpha_i \bigl(y_i(w^T x_i + b) - 1\bigr) = 0$에서, $\alpha_i > 0$이면 해당 데이터는 서포트 벡터이고, $\alpha_i = 0$이면 서포트 벡터가 아니므로 Hyperplane 구축에 영향을 미치지 않는다. 이것이 SVM이 이상치(outlier)에 강인한 이유이기도 하다.

### [개념] Q6. 커널 트릭(Kernel Trick)의 개념과 사용 이유를 서술하고, 대표적인 커널 함수 4가지를 쓰시오.

커널 트릭은 원래 공간에서 선형 분리가 불가능한 데이터를 Feature Map $\Phi$로 고차원 공간에 매핑한 뒤, $\Phi(X)$와 $\Phi(Y)$를 명시적으로 각각 계산하지 않고 커널 함수 $K(X,Y) = \langle \Phi(X), \Phi(Y) \rangle$를 통해 암묵적으로 내적을 바로 계산하여 연산 효율을 높이는 기법이다. 대표적 커널 함수는 다음과 같다.

```math
\begin{aligned}
\text{Linear: }& K(x_1, x_2) = \langle x_1, x_2 \rangle \\
\text{Polynomial: }& K(x_1, x_2) = \left(a\langle x_1, x_2 \rangle + b\right)^d \\
\text{Sigmoid: }& K(x_1, x_2) = \tanh\left(a\langle x_1, x_2 \rangle + b\right) \\
\text{Gaussian(RBF): }& K(x_1, x_2) = \exp\left(-\frac{\|x_1 - x_2\|_2^2}{2\sigma^2}\right)
\end{aligned}
```

### [수식] Q7. 커널 매핑의 예시를 통해 커널 함수가 Feature Map의 내적과 동치임을 보이시오.

2차원 입력 $X = (x_1, x_2)$, $Y = (y_1, y_2)$에 대해 Feature Map을 다음과 같이 정의한다.

```math
\Phi(X) = (x_1^2, x_2^2, \sqrt{2}\,x_1x_2),\qquad
\Phi(Y) = (y_1^2, y_2^2, \sqrt{2}\,y_1y_2)
```

Feature Map의 내적을 직접 계산하면,

```math
\langle \Phi(X), \Phi(Y) \rangle = x_1^2 y_1^2 + x_2^2 y_2^2 + 2x_1x_2y_1y_2
```

한편, 커널 함수 $K(X,Y) = \langle X, Y \rangle^2$를 계산하면,

```math
\langle X, Y \rangle^2 = (x_1y_1 + x_2y_2)^2 = x_1^2y_1^2 + x_2^2y_2^2 + 2x_1x_2y_1y_2
```

따라서 $K(X,Y) = \langle X, Y \rangle^2 = \langle \Phi(X), \Phi(Y) \rangle$이므로, 고차원에서 $\Phi$를 명시적으로 계산하지 않고도 커널 함수를 통해 동일한 내적 값을 효율적으로 얻을 수 있다.

### [개념] Q8. 군집화(Clustering)의 개념과 좋은 군집화의 기준, 그리고 분류(Classification)와의 차이를 서술하시오.

군집화는 유사한 속성을 갖는 데이터를 묶어 전체 데이터를 몇 개의 군집으로 나누는 비지도 학습 방법이다. 좋은 군집화의 기준은 동일 군집 내 데이터는 서로 유사할수록 좋고(intra-class similarity 최대화), 상이한 군집 간 데이터는 서로 다를수록 좋다(inter-class dissimilarity 최대화)는 것이다. 분류는 사전 정의된 범주가 있는 지도 학습이고, 군집화는 사전 정의된 범주 없이 최적의 그룹을 찾아가는 비지도 학습이라는 점에서 근본적으로 다르다.

### [개념] Q9. 군집화 알고리즘의 3가지 유형(계층적, 분리형, 분포 기반)을 각각 설명하시오.

계층적 군집화는 개체들을 가까운 집단부터 차근차근 묶어 나가는 방식으로 덴드로그램(Dendrogram)을 생성하며, 사전에 군집 수를 정하지 않아도 수행 가능하다. 분리형 군집화는 전체 데이터 영역을 특정 기준에 의해 동시에 구분하며, 각 객체들이 사전에 정의된 개수의 군집 중 하나에 속하게 된다(대표: K-means). 분포 기반 군집화는 데이터의 분포를 기반으로 높은 밀도를 갖는 세부 영역들로 전체 영역을 구분하는 방식이다(대표: DBSCAN).

### [수식] Q10. K-means 군집화의 목적 함수를 수식으로 쓰고, 알고리즘 절차와 단점 3가지를 서술하시오.

K-means의 목적 함수는 다음과 같다.

```math
\arg\min \sum_{i=1}^{K} \sum_{x \in C_i} \|x - c_i\|^2
```

여기서 $K$는 군집 수, $C_i$는 $i$번째 군집, $c_i$는 해당 군집의 중심(centroid)이다. 절차는 (1) $K$개의 중심을 임의로 생성하고, (2) 생성된 중심을 기준으로 모든 데이터에 가장 가까운 군집을 할당하며, (3) 각 군집의 중심을 다시 계산하고, (4) 중심이 변하지 않을 때까지 (2)-(3)을 반복한다.

단점은 서로 다른 크기의 군집을 잘 찾아내지 못하고, 서로 다른 밀도의 군집을 잘 찾아내지 못하며, 지역적 패턴이 존재하는 군집을 판별하기 어렵다는 것이다.

### [수식] Q11. Silhouette 통계량의 수식을 쓰고, 이 값의 의미와 최적 군집 수 선정 시 유의점을 서술하시오.

Silhouette 통계량의 수식은 다음과 같다.

```math
\begin{aligned}
s(i) &= \frac{b(i) - a(i)}{\max\{a(i), b(i)\}},\qquad -1 \le s(i) \le 1 \\
\bar{s} &= \frac{1}{n}\sum s(i)
\end{aligned}
```

a(i)는 i번째 데이터와 같은 군집 내 모든 데이터 사이의 평균 거리(작을수록 유사한 데이터가 잘 모여 있다는 의미)이고, b(i)는 i번째 데이터와 다른 군집 내 모든 데이터 사이의 최소 거리(클수록 서로 다른 데이터가 잘 흩어져 있다는 의미)이다. s값이 1에 가까울수록 군집화가 좋고 −1에 가까울수록 나쁘며, 일반적으로 0.5보다 크면 타당한 군집 결과로 판단한다. K=2인 경우 통계량이 최선인 경우가 많으므로, 차순위 K를 선정하는 것이 일반적이다.

### [개념] Q12. DBSCAN의 핵심자료(core point), 주변자료(border point), 잡음자료(noise point)의 정의와 파라미터 ε, M의 역할을 서술하시오.

핵심자료(core point)는 ε-neighborhood가 M개 이상의 데이터를 포함하는 자료이고, 주변자료(border point)는 핵심자료는 아니지만 ε-neighborhood에 핵심자료를 포함하는 자료이며, 잡음자료(noise point)는 핵심자료도 주변자료도 아닌 자료이다. ε은 이웃의 반경을 정의하며 너무 작으면 많은 데이터가 잡음으로 분류되고 너무 크면 군집 수가 적어지고, M은 핵심자료 판별의 최소 데이터 수로 일반적으로 "특성 변수 개수+1"을 사용한다.

### [개념] Q13. K겹 교차 검증(K-fold Cross Validation)의 절차와 홀드아웃 방법 대비 장점을 서술하시오.

### [개념] Q14. 과대적합(Overfitting)과 과소적합(Underfitting)의 개념과 각각의 해결 방법을 서술하시오.

과대적합은 모델이 학습 데이터에 너무 잘 맞지만 일반화(generalization) 성능이 떨어지는 현상으로, 해결 방법은 학습 데이터 추가 수집, 규제(regularization) 값 높이기, 학습 데이터 잡음 줄이기(오류 수정 및 이상치 제거)이다. 과소적합은 모델이 너무 단순하여 데이터에 내재된 구조를 학습하지 못하는 현상으로, 해결 방법은 파라미터가 더 많은 모델 선택, 규제 값 줄이기, 과적합 이전까지 충분히 학습하기이다.

### [개념] Q15. 의사결정나무(Decision Tree)의 개념과 장단점을 서술하시오.

의사결정나무는 학습 데이터에 내재된 패턴을 분석하여 If-then 규칙의 나무 구조로 표현하고, 새로운 데이터를 예측·분류하는 지도 학습 모델이다. 장점은 나무 구조로 모델 해석이 용이하고 의사결정 과정 설명이 가능하며, 데이터의 통계적 가정이 불필요하고, 중요한 변수 선택에 유용하다는 점이다. 단점은 좋은 모형을 만들기 위해 많은 데이터와 시간이 필요하고, 데이터 변화에 민감하며(domain gap이 작아야 함), 선형 구조형 데이터 예측 시 복잡해진다는 점이다.

### [개념] Q16. 의사결정나무의 재귀적 분할 알고리즘 CART, C4.5, CHAID의 차이를 불순도 지표, 분리 방식, 가지치기 방식 측면에서 비교하시오.

CART는 Gini index를 불순도 지표로 사용하며 이진 분할(Binary split)을 수행하고, 완전 모형 생성 후 교차 타당도 기반 가지치기를 한다(학습 데이터로 나무 생성, 검증 데이터로 가지치기). C4.5는 엔트로피(Entropy) 기반 정보 이익(Information Gain)과 정보 이익 비율(Information Gain Ratio)을 사용하며 다중 분할(Multi-way split)을 수행하고, 학습 데이터만 이용하여 나무 성장 및 가지치기를 수행한다. CHAID는 카이제곱 통계량(Chi-square statistic)을 사용하며 다중 분할을 수행하되, 가지치기 없이 최적 모형을 직접 개발한다(불순도 최소 감소량 조건으로 성장을 중단).

### [수식] Q17. CART에서 사용하는 Gini Index의 수식을 쓰고, 예제를 통해 계산 과정을 보이시오.

Gini Index의 수식은 다음과 같다.

```math
\mathrm{G.I.}(A) = \sum_{i=1}^{d} R_i \cdot \left(1 - \sum_{k=1}^{m} p_{ik}^2\right)
```

여기서 $R_i$는 $i$번째 자식 노드의 데이터 비율, $p_{ik}$는 $i$번째 노드에서 클래스 $k$의 비율이다. 값이 낮을수록 노드의 순도가 높아 분류가 잘 된 것이다.

예제: 충성 고객(LC) 5명, 탈퇴 고객(CC) 5명을 성별로 분류 시, 루트 노드의 불순도는 $G_{\mathrm{root}} = 1 - (5/10)^2 - (5/10)^2 = 0.5$이다. 남자 노드(LC:5, CC:1)의 불순도는 $G_{\mathrm{male}} = 1 - (5/6)^2 - (1/6)^2 = 0.278$, 여자 노드(LC:0, CC:4)의 불순도는 $G_{\mathrm{female}} = 0$이므로,

```math
G_{\mathrm{gender}} = \left(\frac{6}{10}\right)\cdot 0.278 + \left(\frac{4}{10}\right)\cdot 0 = 0.167
```

결혼유무로 분류할 때는 $G_{\mathrm{marriage}} = 0.48$이므로, 성별 분류 $(0.167)$가 결혼유무 분류 $(0.48)$보다 Gini Index가 낮아 더 좋은 분리 기준이다.

### [수식] Q18. C4.5에서 사용하는 엔트로피(Entropy), 정보 이익(Information Gain), 정보 이익 비율(Information Gain Ratio)의 수식을 각각 쓰고, 정보 이익 비율이 도입된 이유를 설명하시오.

엔트로피, 정보 이익, 정보 이익 비율의 수식은 각각 다음과 같다.

```math
\begin{aligned}
E(S) &= -\sum p_k \log_2(p_k) \\
IG(A) &= E(\mathrm{Root}) - E(A) \\
IV(A) &= -\sum \left(\frac{n_i}{n}\right)\log_2\left(\frac{n_i}{n}\right) \\
IGR(A) &= \frac{IG(A)}{IV(A)}
\end{aligned}
```

정보 이익만 사용하면 가지 수가 많을수록(다중 분할) 정보 이익이 높아지는 편향이 존재한다. 이를 보완하기 위해 IV(Intrinsic Value)를 도입하여 가지가 많으면 감점하는 정규화 효과를 부여한 것이 정보 이익 비율이다.

### [개념] Q23. 군집화의 유사도 척도 중 유클리디언 거리, 맨하탄 거리, 마할라노비스 거리의 차이를 설명하시오.

유클리디언 거리(Euclidean Distance)는 두 관측치 사이의 직선 거리(L2)로 가장 일반적으로 사용되며, 맨하탄 거리(Manhattan Distance)는 택시 거리(L1)로 각 차원의 절대 차이를 합산한 거리이다. 마할라노비스 거리(Mahalanobis Distance)는 변수 내 분산과 변수 간 공분산을 모두 반영하여 공분산 행렬의 역행렬을 이용해 거리를 계산하며, 데이터의 공분산 행렬이 단위행렬(identity matrix)인 경우 유클리디언 거리와 동일하다.

### [수식] Q24. 마할라노비스 거리의 수식을 쓰고, 유클리디언 거리와의 관계를 설명하시오.

마할라노비스 거리의 수식은 다음과 같다.

```math
d_{\mathrm{mahalanobis}}(X, Y) = \sqrt{(X - Y)^T \Sigma^{-1} (X - Y)}
```

여기서 $\Sigma^{-1}$은 공분산 행렬(covariance matrix)의 역행렬이다. 만약 $\Sigma$가 단위행렬(identity matrix)이면 $\Sigma^{-1}$도 단위행렬이므로 $(X-Y)^T I (X-Y) = (X-Y)^T (X-Y) = \|X-Y\|_2^2$가 되어 유클리디언 거리와 동일해진다. 마할라노비스 거리는 변수 간 상관관계를 반영할 수 있다는 점에서 유클리디언 거리보다 일반적인 거리 척도이다.

### [수식] Q25. 군집화 평가 지표인 SSE(Sum of Squared Error)의 수식을 쓰고, 최적 K 결정 방법을 설명하시오.

SSE의 수식은 다음과 같다.

```math
\mathrm{SSE} = \sum_{i=1}^{K} \sum_{x \in C_i} \mathrm{dist}(x, c_i)^2
```

여기서 $K$는 군집 수, $C_i$는 $i$번째 군집, $c_i$는 해당 군집의 중심이다. SSE는 군집 내 거리를 최소화하는 지표이며, $K$가 증가할수록 SSE는 감소한다. 다양한 군집 수에 대해 SSE를 계산한 뒤, SSE 감소율이 유의미하게 낮아지는 Elbow point에서 최적의 군집 수 $K$를 결정하는 것이 일반적이다.

Neuron은 입력값들의 가중합(weighted sum)에 bias를 더한 뒤 활성화 함수를 적용하는 기본 연산 단위이다. Layer는 여러 뉴런을 수직적으로 묶은 구조로, 입력 벡터를 다른 표현 공간의 벡터로 변환하는 역할을 한다. Activation Function은 뉴런 출력에 비선형성을 부여하여, 네트워크 전체가 단순한 affine mapping으로 축소되는 것을 방지하고 복잡한 함수 근사를 가능하게 한다.

### [수식] Q5. 활성화 함수 $\sigma$가 affine 변환일 경우, 다층 MLP가 단일 affine mapping으로 축소됨을 수식으로 보이시오.

다층 퍼셉트론의 한 층을 다음과 같이 두자.

```math
x^{(l)} = \sigma \left( W^{(l)} x^{(l-1)} + b^{(l)} \right)
```

만약 활성화 함수 $\sigma$가 affine 함수라면,

```math
\sigma(z) = Az + c
```

이므로,

```math
\begin{aligned}
x^{(l)}
&= A\left( W^{(l)} x^{(l-1)} + b^{(l)} \right) + c \\
&= \left( AW^{(l)} \right) x^{(l-1)} + \left( Ab^{(l)} + c \right) \\
&= W' x^{(l-1)} + b'
\end{aligned}
```

즉 한 층이 다시 affine 변환이 된다. 이를 여러 층에 대해 재귀적으로 적용하면,

```math
\begin{aligned}
x^{(l)}
&= W' x^{(l-1)} + b' \\
&= W'' x^{(l-2)} + b'' \\
&= \cdots \\
&= W''' x^{(0)} + b'''
\end{aligned}
```

따라서 아무리 많은 층을 쌓아도 전체 네트워크는 결국 하나의 affine mapping과 동일하다. 그러므로 다층 신경망이 선형 모델 이상의 표현력을 가지려면 비선형 활성화 함수가 반드시 필요하다.

### [개념] Q6. Sigmoid, Tanh, ReLU 활성화 함수 각각의 장단점을 vanishing gradient 및 dying neuron 관점에서 비교 서술하시오.

Sigmoid는 출력 범위가 `(0,1)`이어서 확률 해석에 유리하고 매끄럽게 미분 가능하지만, 입력 절댓값이 커지면 기울기가 거의 0이 되어 vanishing gradient 문제가 심해진다. Tanh는 출력이 `(-1,1)` 범위이고 zero-centered라는 장점이 있어 sigmoid보다 학습이 다소 안정적이지만, 양 끝단에서 여전히 기울기가 0에 가까워져 vanishing gradient가 발생한다. ReLU는 양수 영역에서 기울기가 1로 유지되어 학습이 빠르고 계산이 단순하지만, 음수 영역에서는 기울기가 0이어서 어떤 뉴런이 계속 음수 입력만 받으면 영구적으로 비활성화되는 dying ReLU 문제가 발생할 수 있다.

### [수식] Q7. Perceptron의 학습 규칙과 Convergence Theorem을 서술하시오.

퍼셉트론의 오분류 샘플 `(x_(n_k), y_(n_k))`에 대한 학습 규칙은 다음과 같다.

```math
w^{(k+1)} = w^{(k)} + y_{n_k} x_{n_k}
```

수렴 정리를 위한 가정은 다음 두 가지이다.

```math
\begin{aligned}
(1)\;& \exists R > 0,\ \forall n,\ \|x_n\| \le R \\
(2)\;& \exists w^*,\ \|w^*\| = 1,\ \exists \gamma > 0,\ \forall n,\ y_n(x_n \cdot w^*) \ge \gamma/2
\end{aligned}
```

즉 모든 데이터의 노름은 $R$ 이하이고, 데이터는 margin $\gamma$를 두고 선형 분리 가능하다고 가정한다.

증명의 핵심은 `w*` 방향 성분의 하한과 전체 노름의 상한을 동시에 구하는 것이다.

```math
\begin{aligned}
w^{(k+1)} \cdot w^* &\ge (k+1)\gamma/2 \\
\Rightarrow \|w^{(k)}\|^2 &\ge k^2 \gamma^2 / 4
\end{aligned}
```

한편 오분류 샘플에 대해서만 갱신되므로,

```math
\|w^{(k+1)}\|^2 \le (k+1)R^2
```

두 부등식을 결합하면,

```math
\begin{aligned}
k^2 \gamma^2 / 4 &\le kR^2 \\
\Rightarrow k &\le 4R^2 / \gamma^2
\end{aligned}
```

따라서 퍼셉트론은 최대 ⌊4R²/γ²⌋회 이내의 업데이트 후 수렴하며, 결국 모든 학습 샘플을 정확히 분류하는 해를 찾는다.

### [수식] Q8. Logistic Regression을 network 관점에서 표현할 때, 입력 (GPA=3.8, Experience=1.2)에 대해 $P(Y=1)$의 계산 과정을 서술하시오. (coefficients: $\beta_0=0.4$, $\beta_1=0.2$, $\beta_2=0.5$)

먼저 선형 결합을 계산하면,

```math
\begin{aligned}
z
&= \beta_0 + \beta_1 \cdot \mathrm{GPA} + \beta_2 \cdot \mathrm{Experience} \\
&= 0.4 + 0.2 \times 3.8 + 0.5 \times 1.2 \\
&= 0.4 + 0.76 + 0.6 \\
&= 1.76
\end{aligned}
```

이 값을 sigmoid 함수에 통과시키면 확률은 다음과 같다.

```math
\begin{aligned}
P(Y=1)
&= \frac{1}{1 + e^{-z}} \\
&= \frac{1}{1 + e^{-1.76}} \\
&\approx 0.85
\end{aligned}
```

네트워크 관점에서 보면 입력 노드 `GPA`, `Experience`가 각각 가중치 $0.2$, $0.5$와 곱해지고, bias $0.4$가 더해진 뒤, 최종적으로 sigmoid 노드를 통과하여 출력 확률 $0.85$를 얻는 1-layer neural network로 해석할 수 있다.

### [수식] Q9. Gradient Descent와 Newton's method의 업데이트 규칙을 수식으로 비교하고, Newton's method를 신경망에 사용하지 않는 이유를 서술하시오.

Gradient Descent의 업데이트 규칙은 다음과 같다.

```math
\theta_{k+1} = \theta_k - \alpha \nabla_\theta L(\theta_k)
```

Newton's method는 2차 Taylor 전개를 이용하여 다음과 같이 갱신한다.

```math
\theta_{k+1} = \theta_k - \left[ \nabla_\theta^2 L(\theta_k) \right]^{-1} \nabla_\theta L(\theta_k)
```

Gradient Descent는 gradient만 사용하므로 계산이 단순하지만, 곡률 정보를 직접 반영하지 못한다. 반면 Newton's method는 Hessian을 사용하여 손실 함수의 곡률까지 반영하므로 더 좋은 방향으로 이동할 수 있다. 그러나 신경망에서는 파라미터 수를 $n$이라 할 때 Hessian이 $n \times n$ 크기를 가지며, 저장 비용은 $O(n^2)$, 역행렬 계산은 $O(n^3)$가 필요하다. 실제 딥러닝 모델은 파라미터 수가 수백만~수억 개이므로 계산량과 메모리 요구량이 비현실적이어서 일반적으로 사용하지 않는다.

### [개념] Q10. Momentum의 직관과 업데이트 규칙을 서술하시오.

Momentum의 핵심 직관은 이전 step들의 이동 방향을 누적하여, 일관되게 같은 방향으로 나타나는 gradient는 가속하고 서로 상충하는 방향 성분은 상쇄시키는 것이다. 이로 인해 지그재그 진동이 줄고, 완만한 방향으로는 더 빠르게 이동할 수 있다.

업데이트 규칙은 다음과 같다.

```math
\begin{aligned}
g_k &= \nabla_\theta L(\theta_k) + \mu g_{k-1} \\
\theta_{k+1} &= \theta_k - \alpha g_k
\end{aligned}
```

여기서 $\mu$는 이전 방향을 얼마나 유지할지 결정하는 momentum 계수이다. 결과적으로 loss surface의 골짜기 방향 진동을 줄이고 수렴 속도를 높이는 효과가 있다.

### [수식] Q11. $s = u^T h$, $h = f(z)$, $z = Wx + b$일 때, chain rule을 이용하여 $\partial s / \partial b$를 유도하시오.

Chain rule을 적용하면,

```math
\frac{\partial s}{\partial b}
=
\left( \frac{\partial s}{\partial h} \right)
\left( \frac{\partial h}{\partial z} \right)
\left( \frac{\partial z}{\partial b} \right)
```

각 항은 다음과 같다.

```math
\frac{\partial s}{\partial h} = u^T,\qquad
\frac{\partial h}{\partial z} = D_{f'(z)},\qquad
\frac{\partial z}{\partial b} = I
```

따라서,

```math
\begin{aligned}
\frac{\partial s}{\partial b}
&= u^T D_{f'(z)} I \\
&= u^T D_{f'(z)} \\
&= u^T \odot f'(z)
\end{aligned}
```

여기서 $D_{f'(z)}$는 대각 원소가 $f'(z)$인 diagonal matrix이고, $\odot$는 Hadamard product(원소별 곱)이다. 즉, 각 bias 성분에 대한 민감도는 해당 hidden unit이 출력에 미치는 영향 $u$와 activation의 국소 기울기 $f'(z)$의 원소별 곱으로 표현된다.

### [수식] Q12. Backpropagation의 forward/backward pass 절차를 서술하고, backward pass에서 오른쪽부터 계산하는 것이 왜 효율적인지 설명하시오.

Forward pass에서는 입력 $x$부터 시작하여 각 층의 선형 결합과 활성화 출력을 순차적으로 계산한다.

```math
\begin{aligned}
z^{(i)} &= W^{(i)} a^{(i-1)} + b^{(i)} \\
a^{(i)} &= f(z^{(i)})
\end{aligned}
```

마지막 층에서 loss $L$을 계산한 뒤, backward pass에서는 출력층부터 입력층 방향으로 gradient를 전파한다. 이를 위해 마지막 층의 오차를

```math
\delta = \frac{\partial L}{\partial z^{(n)}}
```

으로 두고, 각 층에 대해 다음을 반복한다.

```math
\begin{aligned}
\frac{\partial L}{\partial \theta_f} &= \left( \frac{\partial f}{\partial \theta_f} \right) \cdot \delta \\
\delta &\leftarrow \left( \frac{\partial f}{\partial x_f} \right) \cdot \delta
\end{aligned}
```

즉 현재 층의 파라미터 gradient를 구한 후, 그 오차를 이전 층으로 전달한다.

오른쪽부터 계산하는 것이 효율적인 이유는 Jacobian들을 모두 명시적으로 만든 뒤 행렬끼리 곱하면 일반적으로 $O(n^3)$ 비용이 들 수 있지만, backpropagation은 항상 Jacobian-vector 곱 형태로 계산하여 각 단계가 $O(n^2)$ 수준으로 줄어들기 때문이다. 또한 중간 결과를 재사용하므로 계산량과 메모리를 크게 절약할 수 있다.

### [개념] Q13. SGD(Stochastic Gradient Descent)가 full-batch GD 대비 갖는 장점을 서술하시오.

Full-batch GD는 한 번의 파라미터 업데이트마다 전체 $N$개 학습 데이터에 대한 loss와 gradient를 모두 계산해야 하므로, 데이터셋이 클수록 한 step의 계산 비용이 매우 크다. 반면 SGD 또는 mini-batch SGD는 $B$개($B \ll N$) 샘플만 사용해 gradient를 추정하므로 한 번의 업데이트가 훨씬 빠르다. 따라서 같은 시간 안에 더 많은 업데이트를 수행할 수 있고, mini-batch에서 발생하는 stochasticity가 local optima나 saddle point 부근에서 탈출하는 데 도움을 줄 수 있다.

### [개념] Q14. Learning rate가 너무 크거나 너무 작을 때 각각 어떤 문제가 발생하는지 서술하고, learning rate decay scheduling의 필요성을 설명하시오.

Learning rate가 너무 크면 최적점을 반복해서 지나치며 loss가 진동하거나 발산할 수 있다. 반대로 너무 작으면 학습은 안정적일 수 있지만 수렴 속도가 매우 느려지고, 비효율적으로 오랜 시간이 걸리거나 좋지 않은 해 근처에 머무를 수 있다. 따라서 초기에는 비교적 큰 learning rate로 빠르게 넓은 영역을 탐색하고, 학습이 진행될수록 learning rate를 점차 줄여 최적점 근처에서 더 정밀하게 수렴하도록 하는 learning rate decay scheduling이 필요하다. 이는 학습 효율과 최종 일반화 성능을 함께 개선하는 데 도움이 된다.

## Part 5. 딥러닝 기초 2

### [개념] Q1. Fully-connected layer 대신 convolutional layer를 사용하는 이유를 설명하시오.

이미지를 fully-connected layer에 직접 입력하면 모든 픽셀이 모든 뉴런에 연결되어 파라미터 수가 폭발적으로 증가한다. 예를 들어 입력이 $128 \times 128 \times 3$이고 출력이 64차원이면 필요한 파라미터 수는 약 300만 개 수준이 된다. 반면 convolutional layer는 이미지의 유용한 특징이 local하다는 점을 활용하여 작은 필터(예: $3 \times 3$)를 전체 이미지에 공유(weight sharing)하므로 파라미터 수를 획기적으로 줄일 수 있다. 예를 들어 같은 상황에서 필요한 파라미터는 $64 \times 27 = 1{,}728$개 수준이다. 또한 동일한 필터를 모든 위치에 적용하므로 위치 변화에 대해 일관된 특징 추출이 가능하며, translation equivariant한 표현 학습에 적합하다.

### [수식] Q2. Convolution layer의 출력 $z[i,j,k]$를 수식으로 표현하고, 각 변수의 의미를 설명하시오.

Convolution layer의 출력은 다음과 같이 표현된다.

```math
z[i,j,k]
=
\sum_{l \in \mathcal{H}}
\sum_{m \in \mathcal{W}}
\sum_{n=0}^{C_{\mathrm{in}}-1}
W[l,m,k,n] \cdot a[i+l,\;j+m,\;n]
+ b[k]
```

여기서 입력 $a \in \mathbb{R}^{H_{\mathrm{in}} \times W_{\mathrm{in}} \times C_{\mathrm{in}}}$, 필터 가중치 $W \in \mathbb{R}^{H_F \times W_F \times C_{\mathrm{out}} \times C_{\mathrm{in}}}$, 편향 $b \in \mathbb{R}^{C_{\mathrm{out}}}$이다. $(i,j)$는 출력 feature map의 공간 좌표, $k$는 출력 채널 인덱스, $(l,m)$은 필터 내부의 상대 좌표, $n$은 입력 채널 인덱스를 뜻한다. 또한

```math
\mathcal{H} = \left\{ -\left\lfloor \frac{H_F}{2} \right\rfloor,\; \ldots,\; \left\lfloor \frac{H_F}{2} \right\rfloor \right\},\qquad
\mathcal{W} = \left\{ -\left\lfloor \frac{W_F}{2} \right\rfloor,\; \ldots,\; \left\lfloor \frac{W_F}{2} \right\rfloor \right\}
```

로 정의된다.

### [수식] Q3. 패딩 없이(valid convolution) 입력 크기 $32 \times 32 \times 3$, 필터 $5 \times 5 \times 6 \times 3$일 때 출력 크기를 계산하시오.

패딩이 없는 valid convolution에서는 필터의 반지름만큼 경계가 잘려 나간다. 필터 크기가 $5 \times 5$이므로 한쪽에서 잘려 나가는 크기는

```math
\frac{H_F - 1}{2} = \frac{5 - 1}{2} = 2
```

이다. 따라서 출력 높이는

```math
H_{\mathrm{out}} = H_{\mathrm{in}} - (H_F - 1) = 32 - 4 = 28
```

이고, 너비도 동일하게

```math
W_{\mathrm{out}} = 32 - 4 = 28
```

이다. 출력 채널 수는 필터 개수인 6이므로 최종 출력 크기는

```math
28 \times 28 \times 6
```

이다.

### [개념] Q4. Zero padding의 장점과 단점을 설명하시오.

Zero padding은 입력 가장자리에 0을 추가하여 convolution 이후에도 feature map의 공간 해상도가 지나치게 줄어들지 않도록 하는 기법이다. 장점은 구현이 간단하고, 층을 여러 번 쌓아도 feature map 크기를 유지하거나 천천히 줄일 수 있어 깊은 네트워크 설계에 유리하다는 점이다. 단점은 경계 부근에서 실제 입력값이 아닌 0과 연산하게 되므로 boundary artifact가 발생할 수 있다는 점이다. 다만 실제 응용에서는 이 문제가 대체로 크지 않아서 가장 널리 사용되는 padding 방식이다.

### [개념] Q5. Stride의 개념과 strided convolution의 장점을 설명하시오.

Stride는 필터를 입력 위에서 슬라이딩할 때 한 번에 이동하는 칸 수를 의미한다. Stride가 1이면 모든 위치에서 연산하지만, stride가 2 이상이면 일부 위치를 건너뛰면서 계산하게 되어 출력 해상도가 줄어든다. Strided convolution의 장점은 해상도 축소와 특징 추출을 하나의 연산으로 동시에 수행할 수 있어 계산량을 줄일 수 있다는 점이다. 또한 일부 연구에서는 convolution 뒤에 별도의 pooling을 두는 방식과 유사한 성능을 보이기도 하므로, pooling 없이도 다운샘플링을 구현할 수 있다.

### [개념] Q6. Receptive field와 Effective Receptive Field(ERF)의 차이를 설명하시오.

Theoretical receptive field는 특정 출력 뉴런이 이론적으로 영향을 받을 수 있는 입력 영역의 전체 크기를 의미한다. 예를 들어 $3 \times 3$ convolution을 두 번 쌓으면 theoretical receptive field는 $5 \times 5$가 된다. 반면 Effective Receptive Field(ERF)는 실제로 출력에 유의미한 영향을 미치는 입력 영역을 의미하며, 보통 $\partial y / \partial x$의 분포를 통해 측정한다. ERF는 이론적 receptive field 전체를 고르게 사용하는 것이 아니라 그 일부에 집중되는 경향이 있으며, 흔히 가우시안에 가까운 분포를 따른다. 따라서 ERF는 네트워크 가중치뿐 아니라 입력 데이터에도 의존한다.

### [수식] Q7. Dilated convolution의 수식을 쓰고, dilation이 receptive field에 미치는 효과를 설명하시오.

Dilated convolution의 출력은 다음과 같이 표현된다.

```math
z[i,j,k]
=
\sum_{l \in \mathcal{H}}
\sum_{m \in \mathcal{W}}
\sum_{n=0}^{C_{\mathrm{in}}-1}
W[l,m,k,n] \cdot a[i+l \cdot d,\; j+m \cdot d,\; n]
+ b[k]
```

여기서 $d$는 dilation rate이다. $d=1$이면 일반 convolution과 동일하고, $d=2$이면 커널 원소 사이 간격이 2가 되어 $3 \times 3$ 커널만으로도 $5 \times 5$ 수준의 receptive field를 갖게 된다. 즉 dilation은 파라미터 수를 늘리지 않으면서 receptive field를 확장할 수 있고, 해상도 손실 없이 더 넓은 문맥 정보를 활용하게 해준다.

### [개념] Q8. Deformable convolution의 핵심 아이디어와 장점을 설명하시오.

Deformable convolution은 고정된 격자 형태의 sampling 위치 대신, 각 필터 위치마다 학습 가능한 offset $\Delta l$, $\Delta m$을 추가하여 샘플링 지점을 유연하게 이동시키는 기법이다. 이 offset은 별도의 convolution layer를 통해 입력 feature map으로부터 예측되며, 필터 원소 수를 $N$이라 하면 offset의 차원은 보통 $2N$이다. 이를 통해 물체의 형태나 자세 변화에 맞춰 receptive field가 적응적으로 변형되므로, 고정된 격자 기반 convolution보다 대상 객체에 더 잘 정렬된 특징 추출이 가능하다.

### [개념] Q9. Transposed convolution의 목적과 checkerboard artifact의 원인 및 해결책을 설명하시오.

Transposed convolution은 저해상도 feature map을 고해상도 feature map으로 upsampling하기 위해 사용되며, 흔히 deconvolution 또는 up-convolution이라고도 부른다. Checkerboard artifact는 커널 크기가 stride로 나누어 떨어지지 않을 때 각 출력 위치가 겹쳐서 계산되는 횟수가 불균일해져 발생하는 격자무늬 패턴이다. 이 현상은 일부 위치는 여러 번 더해지고 일부 위치는 적게 더해지는 overlap 불균형 때문에 생긴다. 이를 완화하는 대표적 방법은 transposed convolution 대신 bilinear upsampling 후 일반 convolution을 적용하는 조합을 사용하는 것이다.

### [수식] Q10. Batch Normalization의 전체 수식(정규화 + learnable parameter)을 쓰고, 각 단계를 설명하시오.

Batch Normalization은 먼저 현재 배치의 평균을 계산한다.

```math
\mu \approx \frac{1}{B} \sum_{j=1}^{B} a_j
```

그 다음 배치 표준편차를 계산한다.

```math
\sigma \approx \sqrt{\frac{1}{B} \sum_{j=1}^{B} (a_j - \mu)^2}
```

마지막으로 정규화한 뒤 학습 가능한 스케일과 시프트를 적용한다.

```math
\bar{a} = \left( \frac{a - \mu}{\sigma} \right)\gamma + \beta
```

여기서 $B$는 배치 크기이고, $\mu$와 $\sigma$는 현재 배치에서 채널별로 계산한 통계량이다. 먼저 activation을 평균 0, 분산 1에 가깝게 정규화하고, 이후 학습 가능한 파라미터 $\gamma$와 $\beta$를 적용하여 네트워크가 필요하면 표현의 스케일과 위치를 다시 조정할 수 있도록 한다.

### [개념] Q11. Internal Covariate Shift 문제와 Batch Normalization이 이를 해결하는 원리를 설명하시오.

Internal Covariate Shift란 학습 과정에서 앞선 layer의 파라미터가 바뀜에 따라 뒤쪽 layer가 받는 입력 분포가 지속적으로 변하는 현상이다. 이 경우 각 layer는 매 step마다 달라지는 입력 분포에 적응해야 하므로 학습이 불안정해지고 수렴이 느려질 수 있다. Batch Normalization은 각 layer의 activation을 배치 단위로 정규화하여 입력 분포를 상대적으로 안정화함으로써 이 문제를 완화한다. 그 결과 더 큰 learning rate를 사용할 수 있고, 학습 속도가 빨라지며, 일부 경우에는 dropout 같은 추가 정규화의 필요성도 줄어든다.

### [개념] Q12. Batch Norm, Layer Norm, Instance Norm, Group Norm의 정규화 축 차이를 설명하시오.

Feature map 텐서를 $(N, C, H, W)$라 하자. Batch Norm은 배치 축 $N$과 공간 축 $(H, W)$에 걸쳐 채널별로 정규화한다. Layer Norm은 단일 샘플 내부의 모든 채널과 공간 축 $(C, H, W)$ 전체에 걸쳐 정규화한다. Instance Norm은 단일 샘플의 단일 채널에 대해서 공간 축 $(H, W)$만 정규화한다. Group Norm은 채널을 여러 그룹으로 나눈 뒤, 각 그룹에 대해 단일 샘플 내부의 그룹별 채널과 공간 축에 걸쳐 정규화한다. 따라서 Group Norm은 배치 크기에 의존하지 않는다는 장점이 있다.

### [수식] Q13. 입력 크기가 큰 경우 gradient 문제가 발생하는 이유를 수식으로 설명하시오.

Linear layer를

```math
z = Wx + b
```

라고 하면, 가중치에 대한 gradient는

```math
\frac{d\mathcal{L}}{dW}
=
\left( \frac{dz}{dW} \right)\left( \frac{d\mathcal{L}}{dz} \right)
=
\delta \cdot x^T
```

로 표현된다. 여기서 입력 벡터 $x$의 각 차원 스케일이 크게 다르면, 예를 들어 $x_1 \gg x_2$인 경우 gradient 역시 특정 방향으로 크게 치우치게 된다. 그러면 loss landscape가 둥근 형태가 아니라 길쭉한 타원체(elongated ellipsoid)에 가까워지고, gradient descent는 최적점으로 곧장 가지 못하고 지그재그로 진동하면서 느리게 수렴한다. 이것이 입력 또는 activation을 standardization해야 하는 중요한 이유이다.

### [개념] Q14. VGG 네트워크에서 $5 \times 5$ conv 대신 $3 \times 3$ conv 2개를 쌓는 이유를 설명하시오.

$3 \times 3$ convolution을 두 번 쌓으면 receptive field는 $5 \times 5$ convolution과 동일하게 $5 \times 5$가 된다. 그러나 중간에 비선형 활성화 함수가 한 번 더 들어가므로 표현력이 더 커진다. 또한 파라미터 수도 더 적다. 채널 수를 $C$라고 하면 $5 \times 5$ convolution은 $25C^2$개의 파라미터가 필요하지만, $3 \times 3$ convolution 두 개는 $2 \times 9C^2 = 18C^2$개만 필요하다. 즉 약 28% 정도 파라미터를 줄이면서도 동일한 receptive field와 더 높은 비선형 표현력을 얻을 수 있다. VGG는 이 원칙을 바탕으로 $3 \times 3$ convolution만 반복하는 homogeneous한 구조를 구성하였다.

### [개념] Q15. ResNet의 skip connection이 학습을 안정화하는 원리를 설명하시오.

Plain network에서는 목표 함수 $H(x)$를 직접 학습해야 하므로, 깊이가 깊어질수록 gradient vanishing 또는 exploding 문제가 쉽게 발생한다. ResNet은 출력을

```math
H(x) = F(x) + x
```

로 정의하여, 전체 함수가 아니라 잔차 함수

```math
F(x) = H(x) - x
```

만 학습하도록 만든다. 이때 미분하면

```math
\frac{\partial H}{\partial x} = \frac{\partial F}{\partial x} + I
```

가 되므로, 가중치가 지나치게 크지 않은 한 Jacobian이 항등행렬 근처에 머물게 된다. 그 결과 gradient가 보다 안정적으로 전파되어 매우 깊은 네트워크도 학습 가능해진다. 이것이 ResNet이 152층 수준까지도 안정적으로 학습될 수 있었던 핵심 이유이다.

### [수식] Q16. Backpropagation에서 Jacobian 곱이 gradient vanishing/exploding을 유발하는 원리를 수식으로 설명하시오.

첫 번째 층의 가중치에 대한 gradient는 chain rule에 의해

```math
\frac{\partial \mathcal{L}}{\partial W^{(1)}}
=
\left( \frac{\partial z^{(1)}}{\partial W^{(1)}} \right)
\left( \frac{\partial a^{(1)}}{\partial z^{(1)}} \right)
\left( \frac{\partial z^{(2)}}{\partial a^{(1)}} \right)
\cdots
\left( \frac{\partial \mathcal{L}}{\partial z^{(n)}} \right)
=
\mathcal{J}_1 \mathcal{J}_2 \cdots \mathcal{J}_n \left( \frac{\partial \mathcal{L}}{\partial z^{(n)}} \right)
```

처럼 여러 Jacobian 행렬의 곱으로 표현된다. 여기서 $\mathcal{J}_i$는 각 layer의 Jacobian이다. 깊은 네트워크에서는 이러한 행렬 곱이 반복되므로, 대부분의 특이값이 1보다 작으면 gradient norm이 점점 0으로 수렴하여 vanishing gradient가 발생하고, 대부분의 특이값이 1보다 크면 gradient norm이 점점 커져 exploding gradient가 발생한다. 따라서 안정적인 학습을 위해서는 각 $\mathcal{J}_i \approx I$가 되는 것이 바람직하며, 이것이 skip connection 설계의 중요한 동기이다.

### [개념] Q17. ResNet의 bottleneck block 구조($1 \times 1 \rightarrow 3 \times 3 \rightarrow 1 \times 1$)의 설계 의도를 설명하시오.

Bottleneck block은 먼저 $1 \times 1$ convolution으로 채널 수를 줄여 계산량을 감소시킨 뒤, $3 \times 3$ convolution으로 공간적 특징을 추출하고, 마지막 $1 \times 1$ convolution으로 다시 채널 수를 복원하는 구조이다. 예를 들어 $256 \rightarrow 64 \rightarrow 256$ 같은 흐름을 사용한다. 이 구조의 목적은 연산량을 크게 늘리지 않으면서 더 깊은 네트워크를 구성하는 것이다. 즉 all-$3 \times 3$ block과 비슷한 계산 복잡도를 유지하면서도 세 층 구조를 사용하여 표현력을 높일 수 있고, 그래서 ResNet-50, ResNet-101, ResNet-152 같은 깊은 모델에서 사용된다.

### [개념] Q18. Inception 모듈에서 $1 \times 1$, $3 \times 3$, $5 \times 5$ conv branch 각각의 역할과 $1 \times 1$ bottleneck의 목적을 설명하시오.

Inception 모듈은 서로 다른 스케일의 특징을 동시에 포착하기 위해 여러 convolution branch를 병렬로 두는 구조이다. $1 \times 1$ convolution branch는 매우 지역적인 pixel-wise 채널 상관관계를 포착하고, $3 \times 3$ branch는 중간 범위의 지역적 특징을, $5 \times 5$ branch는 더 넓은 영역의 문맥 정보를 포착한다. 이때 $3 \times 3$과 $5 \times 5$ branch 앞에 $1 \times 1$ bottleneck convolution을 배치하면 입력 채널 수를 먼저 줄일 수 있으므로, 계산량과 파라미터 수를 크게 줄이면서도 네트워크의 폭과 깊이를 동시에 확장할 수 있다.

### [개념] Q19. Max pooling의 역할과 CNN에서의 필요성을 설명하시오.

Max pooling은 feature map의 작은 영역(예: $2 \times 2$)에서 채널별 최대값만 남기고 나머지를 버려 공간 해상도를 줄이는 연산이다. CNN에서 convolution만 반복하면 feature map 크기가 계속 크게 유지되어 계산량이 증가하고, 최종적으로 하나의 예측 출력으로 집약하기가 어려워진다. Max pooling은 해상도를 점진적으로 줄여 계산량을 낮추고, 중요한 반응만 유지하도록 하며, 작은 위치 변화에 대해 어느 정도의 translation invariance를 부여한다.

### [개념] Q20. Conv-BN-ReLU 순서가 de facto standard인 이유와 Pre-activation(BN→ReLU→Conv) 방식의 차이를 설명하시오.

Conv-ReLU-BN 순서에서는 ReLU를 지난 출력이 항상 비음수 쪽으로 치우친 분포를 가지므로, 그 뒤에 오는 BN이 비대칭 분포를 다시 정규화해야 한다. 반면 Conv-BN-ReLU 순서에서는 convolution의 선형 출력이 먼저 정규화되므로, activation의 스케일과 bias가 더 안정적으로 조정되고 이후 layer에 자연스럽게 전달된다. 그래서 Conv-BN-ReLU는 실무와 연구에서 de facto standard로 자리 잡았다. 한편 Pre-activation 방식인 BN $\rightarrow$ ReLU $\rightarrow$ Conv는 He et al. (ECCV 2016)이 제안한 구조로, skip connection 경로가 순수한 identity mapping에 더 가깝게 유지되어 gradient 흐름이 한층 원활해진다. 이 때문에 매우 깊은 ResNet 계열에서 더 안정적인 학습이 가능하다.
